# Card Game Platform

A multiplayer online card game platform with Blackjack and Poker support.

## Features

- User authentication with invite codes
- Real-time multiplayer with Socket.io
- Blackjack game engine
- Room-based matchmaking
- Quick match feature
- Multiple frontend support (Web, planned: Mobile, TUI)

## Tech Stack

- **Backend**: Node.js + Express + Socket.io + SQLite
- **Frontend**: React + Vite + Tailwind CSS + Zustand
- **Database**: SQLite (better-sqlite3)

## Getting Started

### Prerequisites

- Node.js 20+
- pnpm

### Installation

```bash
pnpm install
```

### Running

```bash
# Start backend (port 4000)
pnpm --filter api dev

# Start frontend (port 5173)
pnpm --filter web dev

# Start both
pnpm dev
```

### First Time Setup

1. Run the seed script to create invite codes:

```bash
cd packages/api
pnpm exec tsx src/db/seed.ts
```

2. Use one of the generated invite codes to register.

### Invite Codes

The system uses invite codes for registration. Initial codes are generated by the seed script.

To create new codes (requires being logged in):

```bash
curl -X POST http://localhost:4000/api/auth/invite-codes \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## API Endpoints

### Authentication

- `POST /api/auth/register` - Register with invite code
- `POST /api/auth/login` - Login
- `GET /api/auth/me` - Get current user

### Rooms

- `GET /api/rooms` - List rooms
- `POST /api/rooms` - Create room
- `GET /api/rooms/:id` - Get room details

## Socket Events

### Client → Server

- `room:list` - Get room list
- `room:create` - Create a room
- `room:join` - Join a room
- `room:leave` - Leave room
- `room:set-ready` - Set ready status
- `game:start` - Start game (owner only)
- `game:action` - Send game action (hit, stand, bet, etc.)
- `quickmatch:join` - Join quick match queue
- `quickmatch:leave` - Leave queue

### Server → Client

- `auth:success` - Authentication successful
- `room:joined` - Joined a room
- `room:player_joined` - Another player joined
- `room:player_left` - Player left
- `room:player_ready` - Player ready status changed
- `game:started` - Game started
- `game:state` - Game state update
- `game:turn` - Your turn
- `game:ended` - Game ended with results
- `quickmatch:found` - Match found
- `error` - Error occurred

## Game Actions (Blackjack)

- `bet` - Place initial bet
- `hit` - Take another card
- `stand` - Keep current hand
- `double` - Double bet and take one card
